# DESIGN.md — BBS Conveyor Design System

This document defines the visual language of the project.

Every AI agent must follow this design system.

Do not invent a different style.

---

# Design Philosophy

BBS Conveyor is a B2B industrial company.

The design should communicate:

- Professional
- Trustworthy
- Premium
- Engineering Precision
- Modern
- Clean

Avoid making the website look like an e-commerce store.

Avoid making the website look like an old industrial catalog.

Think of companies like:

- Siemens
- Schneider Electric
- ABB
- Caterpillar
- Stripe
- Linear
- Vercel

---

# Overall Feel

The website should feel

- spacious
- minimal
- premium
- technical
- easy to scan

Every section should breathe.

Use white space generously.

Never make pages visually crowded.

---

# Color Palette

Primary

Industrial Blue

Secondary

Dark Slate

Accent

BBS Gold

Neutral

White

Light Gray

Border Gray

Never overuse the accent color.

Gold should be used only for:

- CTA
- active state
- icons
- highlights

Never use gold as a large background.

---

# Typography

Style

Modern

Readable

Professional

Hierarchy

H1

Large

Bold

Simple

H2

Bold

Comfortable spacing

Body

Readable

Maximum width around 70 characters.

Never use decorative fonts.

---

# Spacing

Always prefer generous spacing.

Section spacing

Large

Card spacing

Comfortable

Component spacing

Consistent

Never stack elements too tightly.

Whitespace is part of the design.

---

# Border Radius

Cards

Medium

Buttons

Medium

Images

Medium

Avoid excessive rounded corners.

Do not create playful UI.

---

# Shadows

Very subtle.

Never use heavy shadows.

The design should rely on spacing instead of shadows.

---

# Icons

Use outline icons.

Consistent stroke width.

Prefer:

- Lucide

Avoid mixing icon libraries.

---

# Images

Use authentic industrial photography.

Preferred:

- Factory
- Conveyor
- Installation
- Engineering
- Warehouse

Avoid

- Generic stock office images
- Fake AI people
- Cartoon illustrations

Photography should feel real.

---

---

# Buttons

Primary

Filled

Accent color

Secondary

Outline

Ghost

Text only

Never use more than three button styles.

---

# Cards

Cards should have

- image
- title
- short description
- CTA

Avoid large paragraphs inside cards.

---

# Product Section

Grid layout.

Each card should display

Image

Product Name

Short Description

View Details

Never display technical specifications inside the card.

---

# Service Section

Focus on business value.

Not features.

---

# Industry Section

Use icons or industrial imagery.

Each industry should explain

How BBS helps that industry.

---

# Blog Section

Card Layout

Thumbnail

Date

Title

Excerpt

Read More

Avoid showing the full article.

---

# CTA Section

Primary CTA

Request Quote

Secondary CTA

Contact Us

---

# Footer

Four columns

Company

Products

Quick Links

Contact

Bottom

Copyright

Social Media

---

# Animation

Animations should be subtle.

Use

- Fade
- Slide
- Scale

Avoid

- Bounce
- Spin
- Flash
- Excessive parallax

Animation should support usability.

Never distract.

---

# Accessibility

Maintain sufficient contrast.

Clickable targets should be large.

Keyboard navigation should work.

Images require alt text.

---

# Responsive Design

Desktop First

Tablet

Mobile

Do not hide important content on mobile.

Reorganize.

Do not remove.

---

# Components

Components should follow a consistent spacing system.

Never create page-specific components if they can be reused.

Preferred

HeroSection

SectionTitle

ProductCard

IndustryCard

ArticleCard

CTASection

CompanyStats

WhyChooseUs

ProjectGallery

# Layout Principles

Every page should follow this flow.

Hero

↓

Introduction

↓

Core Content

↓

Supporting Content

↓

CTA

↓

Footer

Break information into digestible sections.

---

# Visual Hierarchy

Every page should immediately answer

Who are we?

↓

What do we offer?

↓

Why choose us?

↓

What industries do we serve?

↓

How can visitors contact us?

---

# Things the Agent Must Never Do

Do not use gradients unless requested.

Do not use glassmorphism.

Do not use neumorphism.

Do not use excessive animations.

Do not use random colors.

Do not use inconsistent spacing.

Do not use multiple design languages.

Do not imitate e-commerce websites.

Always prioritize clarity over decoration.

The design should feel like a premium engineering company.
